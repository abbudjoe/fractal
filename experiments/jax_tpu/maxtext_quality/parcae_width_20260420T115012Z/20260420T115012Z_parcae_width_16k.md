# Parcae Width-Scale 16k Rung

Date: 2026-04-20

This run scaled the fixed-policy Parcae proof-ladder shape from the prior
`d_model=128` rung to `d_model=192` while preserving the same data, tokenizer,
8-layer schedule, middle-band Parcae loop, step budget, eval cadence, and seed.

The promotion question was whether the RGRP-control edge survives model-size
scaling after passing the three-seed `d_model=128` replication gate.

## Interpretation

- The width-scale run passed.
- `parcae-rgrp-control` beat attention by `0.073` eval loss and B(x) by
  `0.041` eval loss at `d_model=192`.
- The incremental RGRP-over-B(x) margin increased relative to the three-seed
  `d_model=128` mean (`0.014`) and relative to seed-42 at `d_model=128`
  (`0.025`).
- B(x) still beat attention, so the Parcae scaffold remains a real control.
- RGRP-control still pays a throughput tax: about `74.7%` of attention median
  steady throughput and about `91.7%` of B(x).
- This is still a TPU/JAX quality proof-ladder result, not a CUDA speed claim.

## Decision

RGRP-control survived the first model-size scale rung. The next scientifically
clean move is to replicate the `d_model=192` result across at least two more
seeds before increasing width again.

If compute budget favors faster iteration, run only `parcae-bx` and
`parcae-rgrp-control` for seeds `7` and `123`, because attention has now been
beaten by both Parcae lanes at this shape. If claim strength is the priority,
carry all three lanes for the same seed set.
