v3a Hybrid Attention

A rendered view of the actual Path 1 architecture: a predictive-core hybrid stack that interleaves local exact-attention layers with a sequence-mixing lane that is either a Rust Mamba-style reference block or our `P2` primitive. Tree retrieval belongs to the broader `v3` story, but it is intentionally not part of this baseline matrix.

Source of Truth

This page follows the explicit `v3a` docs now present in the repo: `v3a-hybrid-attention-plan`, `v3a-rust-mamba-baseline-design`, and `v3a-p2-primitive-contract`.

Fixed Path 1 Matrix

  • `A`: eight local exact-attention layers.
  • `A + M`: `A-S-A-S-A-S-A-S`, where `S` is the Rust Mamba reference lane.
  • `A + P2`: `A-P2-A-P2-A-P2-A-P2`, where `P2` is our contender block.
  • Shared phase-1 envelope: width `128`, heads `4`, local window `256`.

Explicitly Out Of Scope

  • No fractal tree retrieval in the Path 1 baseline.
  • No sealed-leaf memory sidecar in the baseline matrix.
  • No routed remote token gathering in the baseline stack.
  • No direct memory-to-logit fusion.

Path 1 Runtime

The hot path remains exact attention. The hybrid question is whether a sequence-mixing block can take the non-attention slots in the stack and preserve quality more efficiently than pure attention everywhere.

What Changed From v2

  • The baseline is no longer tree-first or retrieval-first.
  • Exact attention stays in the predictive core instead of being treated as a rescue patch.
  • The comparison target is a Mamba-family slot inside a hybrid stack.
  • `P2` now competes for that same slot under a matched budget.

What The Diagram Means

  • `A` blocks carry exact local token interaction.
  • `S` blocks are the Rust Mamba-style reference sequence mixer.
  • `P2` blocks are our improved primitive with separate latent state and emitted output.
  • The broader `v3` line may later add cold memory back, but Path 1 isolates the predictive-core question first.
