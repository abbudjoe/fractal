Native Internal Search

A rendered sketch of a model that tries to unify linear reasoning and graph-like search inside one runtime: recurrent multiscale state for cheap persistence, selective attention for exact retrieval, and a bounded latent frontier for multiple live candidate states.

Core Thesis

Internal search likely needs more than attention alone or recurrence alone. This sketch combines both, then adds an explicit latent frontier so the model can hold, compare, and collapse multiple candidate solution states.

Why This Shape

  • Use fractal state-space layers for cheap multiscale persistence.
  • Use selective attention only when exact retrieval earns its cost.
  • Keep search branches latent instead of serializing them into tokens.
  • Verify and halt internally before committing to the next output token.

Interpretation

  • `CoT mode`: frontier width stays near one and halts quickly.
  • `Search mode`: frontier expands, retrieves, prunes, and loops.
  • Both behaviors come from one runtime, not separate model families.

Architecture Sketch

The key move is to treat graph-like search as an internal bounded loop over latent candidates instead of an external chain of expensive full model calls.

Why It Could Be Efficient

  • Branch state stays latent rather than becoming token overhead.
  • Multiscale recurrent state can preserve search context cheaply.
  • Attention becomes sparse and targeted instead of global by default.
  • Halting can happen inside the block loop instead of in a harness.

What Still Has To Be Proven

  • The frontier must hold distinct hypotheses, not near-duplicates.
  • Pruning must avoid premature collapse.
  • The verifier must correlate with actual downstream quality.
  • Training has to reward useful search rather than verbose drift.
